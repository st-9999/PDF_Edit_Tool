# CLAUDE.md

## ドキュメント更新ルール

実装作業の完了時に、以下のドキュメントを必ず更新すること：

1. **`docs/CHANGELOG.md`** — 変更内容を Conventional Commits 形式で追記（Added / Changed / Fixed / Removed）
2. **`docs/SESSION_SUMMARY.md`** — セッションごとの作業サマリを追記。以下のセクションを必ず含めること：
   - **実施内容** — 何を実装・変更したか
   - **作成ファイル** — 新規作成したファイル一覧
   - **変更ファイル** — 既存ファイルへの変更（ある場合）
   - **計測結果** — ビルド結果、テスト通過数、マイグレーション結果など数値で報告
   - **Risks/TODO** — 既知の制約事項、未対応事項、本番デプロイ前に必要な作業、環境変数の設定要件など。なければ「なし」と明記
   - **次ステップ** — 次に着手すべき作業

## envファイルの取り扱い（セキュリティ）

- `.env` ファイルおよび環境変数ファイル（`.env.*`）は絶対にGitHubへpushしないこと
- Claude（AI）は `.env` ファイルを読み書きしないこと
- `.env` ファイルの内容をコード内やログにハードコードしないこと
- AIの応答・出力にAPIキー・シークレット等の秘密情報の値を含めないこと
- 環境変数の設定例を示す場合は、必ずプレースホルダー（例: `your-api-key-here`）を使用すること
- `.env.example` には変数名とプレースホルダーのみ記載し、実際の値は絶対に含めないこと

## ハードコード禁止

- APIキー、シークレット、パスワード等の秘密情報をソースコードに直接記述しないこと
- URL、ポート番号、DB接続文字列などの環境依存値は環境変数または設定ファイルで管理すること
- マジックナンバーやマジックストリングは定数として定義し、意味のある名前を付けること
- テストを通すためだけのハードコーディングは絶対に禁止

## 作業ファイルの配置ルール

- スクリーンショット、デバッグ用画像、一時的な作業ファイルをプロジェクトルートに直接保存しないこと
- これらのファイルは専用フォルダ（例: `tmp/`）に保存し、`.gitignore` に追加すること
- ルートディレクトリにはプロジェクト設定ファイルとツールが要求するファイルのみ配置すること
